## 운영/병합 주의
- WAS source of truth는 파일이 아니라 DB row다.
  - `tb_variable_counterparties.prompt`
  - `use_counterparty_prompt`
  - `only_pending`
  - `designated_password`
  - `delivery_type`
- 아래 WAS 파일은 명시 지시 없이는 수정하지 않는다.
  - `src/app/common/document/parser.py`
  - `src/app/config/settings.py`
  - `.env`
- `only_pending`은 추출 규칙이 아니라 최종 출력 직전 후처리 계약이다.
- `DocumentLoadTaskPayload` shape와 외부 payload schema는 이번 작업 범위에서 바꾸지 않는다.
- `reason_code + reason_summary`는 내부 메타데이터다.
- `*_llm_metrics.json`은 내부 debug artifact다.
- local standalone guidance는 file-based helper를 사용하지만, WAS 운영 guidance는 DB row를 source of truth로 유지한다.
- 프로젝트 전체 clean deployment verdict 기준 known blocker:
  - `app/common/tasks/system_tasks.py`는 `system_cleanup_tasks.yaml`을 찾지만 실제 파일은 `system_cleanup_task.yaml`이다.
  - 결과적으로 `cleanup_events`가 startup 시 등록되지 않는다.
  - 현재 WAS 병합 대상 밖이므로 문서화만 하고 임의 수정하지 않는다.
- 모든 작업은 진행 중 상태를 1분 간격으로 반드시 보고한다.
  - 진행상황 보고에는 현재 단계, 완료/잔여 범위, 신규 실패 또는 차단 여부를 포함한다.
  - 긴 테스트, 병합, 검수, 데이터 추출 배치 작업도 예외 없이 따른다.
