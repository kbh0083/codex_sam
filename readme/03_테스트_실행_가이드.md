## 로컬 테스트
- 로컬 deterministic/unit 테스트는 로컬 repo에서 로컬 venv로 실행한다.
  ```bash
  cd /Users/bhkim/Documents/codex_prj_sam_asset
  .venv/bin/python -m unittest \
    tests.test_service_guard \
    tests.test_output_contract \
    tests.test_counterparty_live_regression \
    tests.test_document_loader_markdown \
    tests.test_extractor_logic \
    tests.test_component -q
  ```
- 로컬 env-gated live regression은 LLM endpoint가 정상일 때만 실행한다.
  ```bash
  cd /Users/bhkim/Documents/codex_prj_sam_asset
  RUN_LIVE_COUNTERPARTY_REGRESSION=1 .venv/bin/python \
    -m unittest tests.test_counterparty_live_regression -q
  ```
- 현재 변경 묶음용 focused live set은 아래 5건을 기준으로 삼는다.
  - `test_cardif_counterparty_prompt_skips_duplicate_xlsx_copy`
  - `test_hanalife_counterparty_prompt_rejects_legacy_pdf_copy`
  - `test_heungkuk_hanais_counterparty_prompt_skips_actual_duplicate_pdf_copy`
  - `test_dongyang_20260318_counterparty_prompt_extracts_expected_html_payload`
  - `test_dongyang_20260413_counterparty_prompt_extracts_expected_html_payload`
- 로컬 live regression을 subset으로 좁힐 때는 unittest test id를 직접 지정한다.
  ```bash
  cd /Users/bhkim/Documents/codex_prj_sam_asset
  RUN_LIVE_COUNTERPARTY_REGRESSION=1 .venv/bin/python \
    -m unittest \
    tests.test_counterparty_live_regression.CounterpartyLiveRegressionTests.test_cardif_counterparty_prompt_skips_duplicate_xlsx_copy \
    tests.test_counterparty_live_regression.CounterpartyLiveRegressionTests.test_hanalife_counterparty_prompt_rejects_legacy_pdf_copy \
    tests.test_counterparty_live_regression.CounterpartyLiveRegressionTests.test_heungkuk_hanais_counterparty_prompt_skips_actual_duplicate_pdf_copy \
    tests.test_counterparty_live_regression.CounterpartyLiveRegressionTests.test_dongyang_20260318_counterparty_prompt_extracts_expected_html_payload \
    tests.test_counterparty_live_regression.CounterpartyLiveRegressionTests.test_dongyang_20260413_counterparty_prompt_extracts_expected_html_payload -q
  ```

- 로컬 데이터 추출 테스트를 새로 수행하면 결과 보고서는 반드시 `test_report`에 작성한다.
  - 보고서에는 케이스별 `retry_counts`와 `elapsed_seconds`를 포함한다.
  - 가능하면 `llm_batches_started`, `stage_retry_invocations`, `transport_retry_attempts`도 sidecar 기준으로 함께 기록한다.
  - 원본 지시서 비교 결과, baseline/comparator 결과, `orders`, `issues`도 함께 기록한다.
  - full 17-case 재실행용 reusable runner는 현재 repo source에 고정돼 있지 않다.
  - 새 harness를 만들 경우 기존 산출물 `/Users/bhkim/Documents/codex_prj_sam_asset/output/debug/local_counterparty_review_20260416_101947`의 manifest/review 구조를 기준으로 만든다.

## WAS 테스트
- WAS LLM 연결 확인은 WAS 테스트 전에 먼저 수행한다.
  ```bash
  curl -sS http://localhost:3900/v1/models
  ```
- WAS syntax/preflight 성격의 빠른 검증은 WAS repo에서 WAS venv로 실행한다.
  ```bash
  cd /Users/bhkim/10_project/01_samsung_asset/samsung_ai_portal_backend
  .venv/bin/python -m py_compile \
    src/app/services/variable_annuity/extract/document_loader.py \
    src/app/services/variable_annuity/extract/extractor.py \
    tests/conftest.py
  ```
- WAS helper regression은 source file이 있을 때만 실행한다.
  ```bash
  cd /Users/bhkim/10_project/01_samsung_asset/samsung_ai_portal_backend
  if [ -f tests/test_variable_annuity_extract_helpers.py ]; then
    PYTHONPATH=src:src/app .venv/bin/python \
      -m pytest -q tests/test_variable_annuity_extract_helpers.py
  else
    echo "tests/test_variable_annuity_extract_helpers.py is missing; restore it before helper regression"
  fi
  ```
- WAS 17-case direct validation은 2026-04-20 harness로 재실행할 수 있다.

  ```bash
  .venv/bin/python output/debug/was_merge_20260420_01/run_was_merge_20260420_01.py
  ```
- WAS pipeline smoke는 WAS venv로 실행한다.
  ```bash

    /Users/bhkim/Documents/codex_prj_sam_asset/output/debug/was_merge_20260420_01/run_pipeline_smoke_20260420_01.py
  ```

- WAS direct/pipeline harness는 기존 `output/debug/was_merge_20260420_01` 산출물을 갱신할 수 있다.
  - 기존 산출물을 보존해야 하면 실행 전에 run root를 복사하거나 새 run id harness를 만든다.
